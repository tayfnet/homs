# تقرير إصلاح شاشة العرض وربط الأسعار

تم فحص شاشة العرض المرفقة وإصلاح مسار جلب الأسعار من Cloudflare D1.

## سبب المشكلة

آخر Worker إنتاجي كان يرجع بيانات شاشة العرض داخل مصفوفة `states`، بينما شاشة العرض كانت تتوقع `state` أو `payload` مباشرة. لذلك كانت أحيانًا لا تقرأ حالة العملات الصحيحة، أو تحتاج تحديثًا يدويًا حتى تظهر البيانات بعد تغيّرها.

## الإصلاحات

- دعم شكل الاستجابة الجديد من Worker: `states[]`.
- دعم الشكل القديم أيضًا: `state` / `payload` حتى لا ينكسر التوافق.
- اختيار حالة العملات بالضبط من المفتاح: `taif-currency-management-module-v1`.
- إضافة بيانات revision/hash/updatedAt داخل الحالة حتى تعرف شاشة العرض متى تغيّرت الأسعار فعليًا.
- جعل التحديث تلقائيًا بجدولة آمنة بدل الاعتماد على التحديث اليدوي.
- منع تداخل طلبات الجلب إذا تأخر اتصال Cloudflare.
- إضافة cache-buster و headers منع الكاش لقراءة آخر سعر دائمًا.
- إضافة backoff عند فشل الاتصال بدل الضغط المتكرر على Worker.
- تحديث Worker حتى يرجع `state` و`payload` مباشرة بالإضافة إلى `states[]` للتوافق الكامل.
- دعم اختبار endpoint عبر GET للتشخيص، مع بقاء شاشة العرض تستخدم POST.

## الفحوصات

- JavaScript syntax: OK
- Worker syntax: OK
- Worker TypeScript diagnostics: 0
- Supabase traces: 0

## طريقة التحديث

1. حدّث Worker الحالي `taif-cloudflare-api` من ملف `cloudflare/worker.js` الموجود في هذه النسخة.
2. ارفع هذه النسخة على مشروع Pages الخاص بشاشة العرض.
3. افتح شاشة العرض واتركها تعمل؛ الأسعار ستتحدث تلقائيًا بدون Refresh يدوي.
